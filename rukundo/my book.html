TOPIC: Web Based Restaurant Management System

INTRODUCTION:

amikus is a restaurant which located near Virunga hotel in Musanze district. Currently they are offering their services to large number of customers daily. Their customer base is growing day by day because of their location and quality of their food. Even though this is a small restaurant, the service they are offering is huge: With the growing customer base, it is difficult to offer services to all of them manually. Therefore, this restaurant requires to move to a computerized solution. This will help to run their operations smoothly and in more efficient manner. PROBLEM: "AMIKUS" is a food store located in Musanze district. Their process is currently manual and time consuming. Customer cannot order food online, before they visit the restaurant. They have to visit the restaurant and place the order. Then they have to wait until the order is ready. There is no visual confirmation that the order was placed correctly. Customers can not ensure about the correctness of their order. Order customization is also difficult because it takes more time. Table reservation is also difficult. This restaurant is very busy in the morning and evening after 6 pm because the restaurant is located in a very attractive place, near Godagama expressway interchange. Therefore, the restaurant is always full. Currently customers have to come to the restaurant to reserve a table. Sometimes customers have to wait long time to reserve a table during busy times. Also, very long queues are there. It is difficult for the cashier to handle such a situation. On the other hand, 2 this situation is difficult for the kitchen also. They are receiving orders one by one. Sometimes kitchen is receiving same food item within separate orders in same time.

SOLUTION: Online system that proposed here will simplify the ordering process and it will be helpful to both restaurant and customer. The proposed system will be developed with interactive menus, pop-up messages etc. for the easiness of the user. Customer can select the required food item with available customizations and the can change the order at any time before checking out online. If order is confirmed, it will display a pop-up message as confirmation to the customer. When the order is placed it's recorded in the database and retrieved in real time manner. Through this option Restaurant Employees are allowed to quickly go through the orders which they have received. Therefore, they can process all orders efficiently and effectively with minimal delays and confusion. GENERAL OBJECTIVE: Proposed system is valuable for both customer and the restaurant because it is simplifying the order processing process. Through better application of daily operations restaurant can increase the efficiency and can offer improved services to the customers. Because almost all processes are manual and time consuming, all the processes should be automated. The Main Objective: To build a web based restaurant management system for "AMIKUS" Restaurant.

PORTABILITY REQUIREMENTS:

 Cross-platform Compatibility: The system should be compatible with major web browsers such as Chrome, Firefox, Safari, and Edge. It should work seamlessly across different operating systems, including Windows, macOS, Linux, and mobile platforms (iOS and Android).

 Responsive Design: The system should be responsive and adaptable to different screen sizes, resolutions, and orientations. This ensures that it can be accessed and used effectively on various devices, including desktop computers, laptops, tablets, and smartphones.

 Internet Connectivity: As a web-based system, it relies on internet connectivity. Therefore, it should be designed to function well in both low and high bandwidth environments. It should gracefully handle intermittent connections and provide offline capabilities

RELIABILITY REQUIREMENTS:  Availability: The system should be highly available, ensuring that it remains accessible to users without significant downtime. This includes reliable web hosting, redundant infrastructure, and proactive monitoring to detect and address any potential issues quickly.
 Performance: The system should be responsive and capable of handling a significant number of concurrent users, especially during peak hours. It should be optimized to provide fast response times, minimizing delays and ensuring a smooth user experience.
 Data Integrity: The system should ensure the integrity of data by implementing robust mechanisms for data storage and backup. This includes regular backups, data replication, and data validation to prevent data loss or corruption

USABILITY REQUIREMENTS:  Intuitive User Interface: The system should have a user-friendly and intuitive interface that is easy to navigate for both restaurant staff and customers. Clear menus, buttons, and labels should be used to minimize confusion and make it easy to perform tasks.

 Responsiveness: The system should be responsive and adapt to different screen sizes and devices, including desktop computers, laptops, tablets, and smartphones. This ensures that users can access and use the system effectively regardless of the device they are using.

 Efficient Task Execution: The system should enable restaurant staff to perform their tasks efficiently and effectively. This includes features such as easy order management, table management, menu updates, inventory tracking, and reporting. Streamlining and automating processes can help save time and reduce errors.

SPACE REQUIREMENTS:  Software Size: The size of the software itself, including the web application files, images, scripts, and any other necessary components, will contribute to the space requirements. This can typically range from a few megabytes to several gigabytes, depending on the system's functionality and the technology stack used for development.

 Database Size: If the restaurant management system includes a database to store information such as menu items, orders, customer data, and inventory, the size of the database will also contribute to the space requirements.
Media Storage: If the system includes features that require storing media files, such as images or videos for menu items or promotional materials, the space requirements will increase accordingly. The amount of media storage needed will depend on factors such as the number of items and the size of the media files being stored.

ORGANIZATION REQUIREMENTS:  User Roles and Permissions: Define different user roles within the system, such as administrators, managers, waitstaff, kitchen staff, and customers. Each role should have specific permissions and access levels to perform their respective tasks. This ensures proper data security, confidentiality, and accountability.

 Hierarchical Structure: Establish a hierarchical structure within the system to reflect the organizational structure of the restaurant. This may include sections for different departments, such as front of house, kitchen, inventory management, and accounting. This structure aids in organizing tasks, responsibilities, and information.
 Centralized Data Management: Implement a centralized database to store and manage all relevant data, including menu items, orders, customer information, inventory, and financial records.

IMPLEMENTATION REQUIREMENTS:  User Interface (UI) Design: Develop an intuitive and user-friendly interface that allows restaurant staff to easily navigate and access the system's features. The UI should provide clear menus, forms, and buttons for adding, editing, and managing various aspects of the restaurant.

 Menu Management: Implement a feature to create, update, and manage the restaurant's menu items. This includes adding item names, descriptions, prices, and categorizing them under different sections (appetizers, main courses, desserts, etc.). The system should also allow for customization options and support images for menu items.
 Ordering and POS Integration: Enable customers to place orders online and integrate the system with the restaurant's point of sale (POS) system. This integration ensures that online orders are seamlessly processed and sent to the kitchen for preparation. It should also update inventory levels in real-time.

EXTERNAL ENVIRONMENT REQUIREMENT:

 Internet Connectivity: A stable and reliable internet connection is essential for web-based restaurant management. It enables real-time communication, data transfer, and access to cloud-based systems. Ensure that the restaurant has a strong internet connection with sufficient bandwidth to handle the required tasks.

 Hardware and Software Compatibility: Verify that the hardware devices (computers, tablets, smartphones) and software systems (operating systems, web browsers) used by the restaurant staff are compatible with the web-based management solution. This ensures seamless integration and smooth operation.
 Security Measures: Protecting sensitive information, such as customer data and financial records, is crucial. Implement appropriate security measures, including encryption, firewalls, secure payment gateways, and regular data backups, to safeguard against cyber threats and data breaches.

PRIVACY REQUIREMENTS:  Data Encryption: All sensitive data, such as customer information, employee details, and financial records, should be encrypted during transmission and storage. Encryption ensures that even if the data is intercepted or accessed by unauthorized individuals, it remains unreadable and secure.

 User Authentication: Implement strong user authentication mechanisms, such as username/password combinations, two-factor authentication (2FA), or biometric authentication, to ensure that only authorized individuals can access the system. This helps prevent unauthorized access to sensitive data.
 Access Control: Implement robust access control mechanisms to restrict user access based on their roles and responsibilities within the restaurant.
SAFETY REQUIREMENTS:  Secure Authentication: Implement robust authentication mechanisms to ensure that only authorized individuals can access the system. Strong passwords, multi-factor authentication (MFA), or biometric authentication can be used to enhance security.
 User Role-Based Access Control: Implement a role-based access control (RBAC) system to define and enforce user privileges based on their roles and responsibilities within the restaurant. This ensures that users can only access the functionalities and data necessary for their specific job functions.
 Input Validation: Implement strict input validation mechanisms to prevent common web-based vulnerabilities such as SQL injection, cross-site scripting (XSS), and cross-site request forgery (CSRF). Validate and sanitize user input to mitigate the risk of malicious attacks.
